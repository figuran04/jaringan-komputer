<h4>with <a href="https://medium.com/u/cc0a3919bebc">Salieri</a></h4><figure><img alt="" src="https://cdn-images-1.medium.com/max/821/1*5l7LrmwT-5ods6nzLCHN2Q.png" /></figure><p>Routing statis adalah metode di mana administrator jaringan menetapkan jalur tetap untuk setiap tujuan dalam jaringan. Hal ini memungkinkan kontrol penuh atas rute yang diambil oleh data, namun membutuhkan pengaturan manual dan perhatian lebih tinggi pada perubahan topologi jaringan. Routing statis banyak digunakan dalam jaringan kecil atau jaringan yang jarang mengalami perubahan, karena pada jaringan yang lebih besar atau dinamis, pengaturan ini bisa memakan waktu lebih lama dan lebih rumit dibandingkan dengan routing dinamis.</p><p>Topologi jaringan sederhana dengan dua router dapat mengilustrasikan cara kerja routing statis, termasuk bagaimana paket data diteruskan dari satu host ke host lain melalui router. Proses ini akan dijelaskan dengan tabel routing dan alur kerja paket data dari Host A ke Host C.</p><h3>Router</h3><p>Router adalah perangkat yang bertugas menghubungkan berbagai jaringan dan menentukan jalur terbaik bagi paket data untuk mencapai tujuan. Dalam routing statis, setiap rute diatur secara manual oleh administrator.</p><h3>Tipe Mekanisme Routing</h3><ol><li><strong>Routing Statis</strong> — Jalur diatur secara manual oleh administrator jaringan.</li><li><strong>Routing Dinamis</strong> — Jalur dipilih secara otomatis menggunakan protokol routing seperti RIP atau OSPF yang mendeteksi topologi jaringan.</li></ol><h3>Routing Table dan Prinsip Kerja Routing</h3><p>Routing atau perutean adalah proses memindahkan packet data dari sumber ke tujuannya. Proses routing membutuhkan routing table sebagai pedoman bagi router untuk menentukan jalur pengiriman packet data agar sampai tujuannnya.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*8AAQzETJJF-CFG52mgqlUQ.png" /></figure><p>Perhatikan skema jaringan dua router pada Gambar 1. Ketika suatu komputer (misal host A) yang tergabung pada LAN 1 akan mengirim pesan ke komputer yang tergabung pada LAN 4 (misal host C) harus melalui dua router yaitu R1 dan R2. Setiap router tersebut harus memiliki routing table agar dapat meneruskan packet data (pesan) dari host A ke host C. Informasi dalam routing table yang terdapat pada R1 dan R2 diperlihatkan pada Tabel 1 dan Tabel 2.</p><h4>Tabel 1. Routing Table R1</h4><a href="https://medium.com/media/deea41fda4008e5ccbb291994d47da06/href">https://medium.com/media/deea41fda4008e5ccbb291994d47da06/href</a><h4>Tabel 2. Routing Table R2</h4><a href="https://medium.com/media/a96f774d1dcd4d434dfd52753d1609c0/href">https://medium.com/media/a96f774d1dcd4d434dfd52753d1609c0/href</a><p>Penjelasan dari informasi yang terdapat di dalam sebuah routing table disajikan pada Tabel 3:</p><h4>Tabel 3. Informasi Dasar Pada Routing Table</h4><a href="https://medium.com/media/51e008d9d95f11631434cd40dfd4d8c5/href">https://medium.com/media/51e008d9d95f11631434cd40dfd4d8c5/href</a><h3>Prinsip Kerja Routing Berdasarkan Gambar</h3><p>Untuk ilustrasi pengiriman paket dari Host A ke Host C, alur kerja routingnya adalah sebagai berikut:</p><ol><li><strong>Router R1</strong> menerima paket data dari Host A dan memeriksa alamat IP tujuan. Alamat tujuan ini menunjukkan jaringan <strong>192.168.30.0/24</strong>, di mana Host C berada.</li><li><strong>Router R1</strong> mencocokkan <strong>destination network</strong> (jaringan tujuan) dari Host C dengan entri di dalam tabel routing R1.</li><li><strong>Router R1</strong> menemukan bahwa jaringan tujuan <strong>192.168.30.0/24</strong> dapat diakses melalui <strong>gateway 192.168.40.2</strong> pada <strong>interface ether2</strong>. R1 kemudian meneruskan paket data ke Router R2 melalui interface tersebut.</li><li><strong>Router R2</strong> menerima paket data dari Router R1 dan memeriksa alamat IP tujuan untuk mengetahui jaringan <strong>192.168.30.0/24</strong>.</li><li><strong>Router R2</strong> mencocokkan <strong>destination network</strong> dari Host C dengan tabel routing R2.</li><li><strong>Router R2</strong> menemukan bahwa jaringan tujuan <strong>192.168.30.0/24</strong> terhubung langsung melalui <strong>interface ether3</strong>. Oleh karena itu, R2 meneruskan paket data ke Host C melalui interface tersebut.</li><li><strong>Host C</strong> menerima paket data dari Host A.</li></ol><h3>Konfigurasi Static Routing</h3><p>Untuk konfigurasi routing statis, berikut adalah perintah dasar yang digunakan pada perangkat Cisco:</p><pre>R1(config)#ip route network [mask] {address | interface}[distance] [permanent]</pre><h4>Tabel 5. Perintah Dasar Konfigurasi Static Routing</h4><a href="https://medium.com/media/feba33124049bcb0d985dd2402999760/href">https://medium.com/media/feba33124049bcb0d985dd2402999760/href</a><h3>Fungsi Router</h3><p>Router berfungsi untuk mengarahkan paket data ke jaringan yang sesuai, menjaga keamanan antar-jaringan, serta memastikan data mencapai tujuan yang tepat. Secara lengkap beberapa fungsi router adalah sebagai berikut:</p><ol><li>Menghubungkan beberapa LAN.</li><li>Mentransmisikan (forwarding) packet data antar jaringan.</li><li>Menghubungkan jaringan lokal ke koneksi DSL (Digital subscriber line) agar mendapatkan keamanan tambahan.</li><li>Menyaring packet data internet yang melintasi jaringan. Menyimpan routing table untuk menentukan rute terbaik antar LAN pada jaringan WAN.</li></ol><p>Dalam konfigurasi routing statis, administrator dapat mengontrol rute mana yang digunakan untuk setiap paket, yang membantu menjaga kestabilan dan keamanan koneksi pada jaringan kecil.</p><h3>PRAKTIKUM</h3><h3>I.
